# Virtual User Agent

## Perspektywa uzytkownika

Jako uzytkownik oczekuje, ze pierwsze wejscie na strone da mi szybki dostep do horoskopu, tarota, numerologii albo artykulow. Nie chce trafic na link sklepu, jesli sklep jest wylaczony. Jesli strona jest w maintenance, chce czytelnej informacji i dostepu do legal/newsletter tam, gdzie to ma sens.

## Typowa sciezka

1. Wejscie na `/`.
2. Wybor horoskopu, tarota lub artykulu.
3. Opcjonalnie rejestracja/logowanie.
4. Przejscie do panelu albo premium.
5. Newsletter lub kontakt, jesli chce zostawic dane.

## Mozliwe punkty frustracji

- Niejasny status premium: otwarte premium vs platna subskrypcja.
- Linki sklepu widoczne mimo wylaczonej funkcji.
- Brak danych contentowych po seedzie albo nieskonczone skeletony.
- Maintenance bez jasnego kontaktu lub ETA.

## Polish summary

Najwazniejsze dla realnego uzytkownika sa spojnosc widocznych funkcji z feature flagami, czytelne stany puste oraz brak obietnicy platnego premium, jesli Stripe nie jest gotowy.
